# Mobile responsive operator UI

**Filed:** Day 12 evening (5 May 2026), Phase 2 deferral
**Source:** PLANNER_PRODUCT_BRIEF.md §4 + §8 Open Questions; brief §3.3.11 brand pass scope
**Phase 2 trigger:** Post-pilot (or Day-18 polish if time permits per brief §8)

## What

Full mobile + tablet responsive layouts for all operator surfaces:
- Calendar grid (week/month/year) on touch devices
- Onboarding wizard on mobile-portrait
- Consolidated calendar with filter bar collapsed
- Popover positioning for thumb-reach
- Touch-target sizing per WCAG

MVP ships desktop-first (1200px+) with no mobile-portrait optimisation. Tablet-landscape probably degrades acceptably.

## Why deferred

Pilot operators are on desks (CS agents on workstations, ops managers on laptops). Mobile-first is a Phase 2 productivity gain — driver-facing apps remain in the SF mobile app, not the Planner.

## When unlocked

Post-pilot. May land earlier if Day-18 polish slot has slack — brief §8 explicitly says "Phase 2 unless time permits Day 18 polish."
